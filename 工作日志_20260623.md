### 数据库列检查与添加
```csharp
private bool ColumnExists(SQLiteConnection conn, string tableName, string columnName)
{
    using var cmd = new SQLiteCommand($"PRAGMA table_info({tableName})", conn);
    using var reader = cmd.ExecuteReader();
    while (reader.Read())
    {
        if (string.Equals(reader[1]?.ToString(), columnName, StringComparison.OrdinalIgnoreCase))
            return true;
    }
    return false;
}

// 使用示例
if (!ColumnExists(conn, "ProductMaterialLibrary", "quantity"))
{
    using var cmd = new SQLiteCommand(
        "ALTER TABLE ProductMaterialLibrary ADD COLUMN quantity REAL", conn);
    cmd.ExecuteNonQuery();
}
```

### 16. 修复柜体多选物料重复显示问题

**问题描述**：
- 在配置柜类对话框中，柜体物料多选时，同一个物料被显示两次，导致小计的数量和价格翻倍。

**根本原因**：
- `MaterialGroupEditorDialog.xaml.cs` 中 `OpenSelector` 方法对 Multiple 模式使用追加模式（不清空直接添加）。
- 程序初始化时会加载数据库中已有的物料。
- 用户再次打开选择框多选物料时，新选择的物料被追加到已有物料后面，导致重复。

**解决方案**：
- 统一 Multiple 和 Single 模式的行为：先清空已有物料，再添加新选择的物料。
- 删除原有的 if-else 分支逻辑。

**修改文件**：
- `Views/MaterialGroupEditorDialog.xaml.cs` - 第 160-174 行

**修改内容**：
```csharp
// 修改前：Multiple 模式追加，Single 模式替换
if (row.Item.SelectionRule == SelectionRuleType.Multiple)
{
    foreach (var m in dlg.SelectedMaterials) row.AddMaterial(m);
}
else
{
    row.ClearMaterials();
    foreach (var m in dlg.SelectedMaterials) row.AddMaterial(m);
}

// 修改后：统一先清空再添加
row.ClearMaterials();
foreach (var m in dlg.SelectedMaterials) row.AddMaterial(m);
```

**完成时间**：2026-06-23 16:20

**测试结果**：✅ 通过 - 多选物料不再重复，小计数量和价格正确
