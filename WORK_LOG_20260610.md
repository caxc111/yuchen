- 修复产品查询中的关键字参数判空逻辑，避免 `keyword.Trim()` 触发空引用风险。
- 调整 `GetMyMaterialCodeByFactoryMaterialCode` 返回类型为可空字符串，以匹配实际数据库返回行为。
- 收敛 `MaterialCodeGenerator` 中分类编码读取与已有编码复用逻辑的 nullable 推断问题。

**补充说明**：
- 这部分修复既消除了编译警告，也提升了数据库读取结果与 C# 空值语义的一致性。
- 编码生成逻辑现在对工厂、一级分类、二级/三级分类的缺失情况有更清晰的失败提示。

### 4. 整理并落地产品编码规则（首版）
- 确认宇辰产品编码品牌前缀采用 `IH`，作为 `INTELHOME` 的统一缩写。
- 确认产品编码整体格式为 `IH-{业态码}-{户型码}-{流水号}`。
- 将原“住宅”业态调整为“公寓”，并新增 `House` 业态，下拉项现为 `公寓 / House / 公区 / 酒店 / 商业`。
- 明确户型编码采用国际化缩写规则，如 `1R1B` 表示一房一卫。
- 在产品管理弹窗中增加首版“生成编码”按钮，可根据业态与户型自动生成产品编码。
- 流水号已接入数据库查询逻辑，当前按“业态码 + 户型码”维度自动取下一个 3 位序号。
- 明确流水号统一使用 3 位数字格式，为后续编码稳定扩展保留空间。

**产品编码规则（首版）**：
- 编码格式：`IH-{业态码}-{户型码}-{流水号}`
- 编码示例：`IH-A-1R1B-001`
- 字段说明：
  - `IH`：宇辰品牌前缀，代表 `INTELHOME`
  - `业态码`：产品所属业态
  - `户型码`：产品所属户型
  - `流水号`：同类产品顺序编号

**业态编码**：
- `A` = 公寓
- `H` = House
- `P` = 公区
- `HT` = 酒店
- `C` = 商业

**户型编码**：
- 采用 `xRyB` 结构
- `R` = Room
- `B` = Bathroom
- 示例：
  - `1R1B` = 一房一卫
  - `2R1B` = 两房一卫
  - `2R2B` = 两房两卫
  - `3R2B` = 三房两卫
  - `4R3B` = 四房三卫
- 非住宅类业态当前使用 `NA` 作为户型占位码

**流水号规则**：
- 统一采用 3 位数字，如 `001`、`002`、`003`
- 不足 3 位时左侧补 `0`
- 当前界面生成功能已按“业态码 + 户型码”维度自动查询下一个可用流水号
- 后续如需跨库或多人并发生成，可再补充唯一性冲突兜底策略

**补充说明**：
- 产品编码仅用于标识产品身份，不绑定工厂编码、物料编码、价格、面积或日期等易变信息。
- 当前已在产品编码输入区域增加首版“生成编码”按钮，可按上述规则自动生成编码。

## 技术要点

- WPF 中孤儿 code-behind 文件会参与编译，但如果缺少对应 XAML，就会直接引发 `InitializeComponent` 相关错误。
- nullable 清理不能只改声明层，还需要同步梳理数据库读取、UI 绑定、事件、构造参数和业务方法签名。
- 对数据库 `NULL` 的处理应与模型语义保持一致，否则会在开启 nullable 后持续产生告警并隐藏运行时风险。
- 先修“会引发真实空引用的点”，再批量收告警，是这类老项目较高效的治理方式。
- 产品编码规则应优先保证稳定、可读、可扩展，避免将易变业务信息直接编码进主编码结构。
