# 工作日志

## 2026-06-30

### Bug修复
- **ProductPartMaterial 类 DrawingNumber 属性重复定义**: 类中 `DrawingNumber` 属性定义了两次导致编译错误
 - 修改文件：`ProductPartMaterial.cs`
 - 状态：已完成

- **子物料图纸编号唯一性检查错误触发**: 复合物料子物料（如 JR3 的柜体 JF1）在选择时错误提示"图纸编号重复"
 - 根因：子物料不需要图纸编号唯一性检查，但系统对所有物料都进行了检查
 - 修复1：在 `MaterialSelectorDialog` 添加 `SkipDrawingNumberCheck` 属性
 - 修复2：在 `MaterialGroupEditorDialog.OpenSelector` 中设置 `SkipDrawingNumberCheck=true`
 - 修改文件：`MaterialSelectorDialog.xaml.cs`、`MaterialGroupEditorDialog.xaml.cs`
 - 状态：已完成

- **单一物料图纸编号唯一性验证缺失**: 复合物料的图纸编号唯一性验证已实现，但普通物料在 DataGrid 中直接编辑时绕过了验证
 - 根因1：`DrawingNumberInDialogTextBox_LostFocus` 方法中没有验证逻辑
 - 根因2：`OkButton_Click` 中批量设置图纸编号时没有验证重复性
 - 修复1：在 `LostFocus` 中添加图纸编号重复性检查，发现重复则清空并提示
 - 修复2：在批量设置前调用 `CheckDrawingNumberExistsInProject` 检查重复
 - 修改文件：`MaterialSelectorDialog.xaml.cs`
 - 状态：已完成

- **添加产品时项目代码智能填充失败**: `GetAllProjectCodes()` 查询列名错误
 - 根因：SQL 查询 `project_code` 列，但表中实际列名是 `product_code`
 - 修复：将 SQL 中的 `project_code` 改为 `product_code`
 - 修改文件：`DbService.cs`
 - 状态：已完成

- **同项目不同产品无法选择相同物料**: XN29-A-1R1B-B 无法选择已在 XN29-A-1R1B-A 中定义的 FF1（LVT）
 - 根因：图纸编号唯一性检查范围错误，同一项目内不应限制
 - 规则：同一项目内物料和图纸编号完全通用，不同项目间不可通用
 - 修复：移除所有图纸编号唯一性检查（同项目内通用）
 - 修改文件：`MaterialSelectorDialog.xaml.cs`、`AddProductMaterialWindow.xaml.cs`
 - 状态：已完成
