## 今日完成工作

### 1. 问题背景：柜类物料选择如何建议

**业务场景**：
- 橱柜/洗衣柜/中岛台等柜类，内部天然包含多个子项（如"柜体+门板+台面+五金+拉手"）。
- 台面材质可能是石英石或大理石，**二选一**，用户需要能选到具体材质。
- 原有模型（部位→部品→物料类型→物料）中，一个柜子被压缩成一个枚举，无法表达内部多子项结构。

**最终选择**：方案 B——**复合物料 / 物料子项**。
- 数据库：新增 MaterialGroups（组合定义）+ MaterialGroupItems（子项模板）+ ProductPartMaterials 扩展列（IsComposite / GroupCode / ItemName / ParentId）。
- UI：下拉选"定制橱柜"→弹出 MaterialGroupEditorDialog→逐子项选物料（复用 MaterialSelectorDialog）→返回 SelectedMaterial（含 Children 集合）→大卡显示子项列表。

### 2. 数据库设计（零迁移成本）

#### 模板定义表

**MaterialGroups**（组合"菜谱"）
```sql
CREATE TABLE MaterialGroups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_code TEXT NOT NULL UNIQUE,  -- CB-001 / XYG-001 / ZDT-001
    group_name TEXT NOT NULL,         -- "定制橱柜" / "洗衣柜" / "中岛台"
    category TEXT,                    -- "柜类"
    description TEXT,
    is_active INTEGER DEFAULT 1,
    created_at TEXT, updated_at TEXT
);
```

**MaterialGroupItems**（菜谱的每道工序）
```sql
CREATE TABLE MaterialGroupItems (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_id INTEGER NOT NULL,
    item_name TEXT NOT NULL,          -- "柜体" / "门板" / "台面" / "五金" / "拉手"
    item_order INTEGER DEFAULT 0,
    material_type TEXT,                -- "板材"/"石英石,大理石,岩板"/"铰链"
    selection_rule TEXT DEFAULT 'Single',  -- Single / SingleOrNone / Multiple
    is_required INTEGER DEFAULT 1,    -- 0=可选 1=必选
    prompt TEXT,
    FOREIGN KEY (group_id) REFERENCES MaterialGroups(id) ON DELETE CASCADE
);
```

#### 落库表扩展

ProductPartMaterials 新增 4 列：
```sql
ALTER TABLE ProductPartMaterials ADD COLUMN is_composite INTEGER DEFAULT 0;  -- 1=主行
ALTER TABLE ProductPartMaterials ADD COLUMN group_code TEXT;                   -- CB-001
ALTER TABLE ProductPartMaterials ADD COLUMN item_name TEXT;                     -- "台面"
ALTER TABLE ProductPartMaterials ADD COLUMN parent_id INTEGER;                  -- 子行→主行 id
```

**核心思路**：复合物料所有数据全部存入 ProductPartMaterials——主行 1 条 + 子行 N 条（parent_id 指向主行）。现有查询/删除 API 零修改。

### 5. SelectedMaterial 扩展（内存模型）

新增 4 属性 + Children 集合：

```csharp
public class SelectedMaterial {
    public bool IsComposite { get; set; }           // 1=主行
    public string GroupCode { get; set; }            // CB-001
    public string ItemName { get; set; }             // 主行=""，子行="台面"
    public int? ParentRef { get; set; }              // 内存父子引用
    public ObservableCollection<SelectedMaterial> Children { get; }

    // 复合物料：递归算子项之和；普通物料：UnitPrice * Quantity
    public decimal TotalPrice {
        get {
            if (IsComposite) {
                decimal sum = 0;
                foreach (var c in Children) sum += c.TotalPrice;
                return sum;
            }
            return UnitPrice * Quantity;
        }
    }
}
```

- 子项集合变化 / 子项 Quantity 变化时，均通过 `_parentForNotify` 向上冒泡 `TotalPrice` 通知，UI 自动刷新。

## 技术要点

- **WPF DataTemplateSelector**：`ItemTemplate` 和 `ItemTemplateSelector` 不能同时存在，必须把两个模板都放入 Resources，再由 Selector 引用。
- **WPF Run 不支持 Visibility**：绑定在 `Run` 上的 `Visibility` 会编译报错，必须去掉或用其他方案（如 `TextBlock` 叠加）。
- **计算属性的 setter**：C# `=>` 表达式体属性是只读的，不能作为赋值目标。若需要可写（如复合物料主行 TotalPrice = 子项之和），改用 `get; set;` + backing field。
- **子项 Quantity 变化向上冒泡**：通过 `_parentForNotify` 引用，在子项 `Quantity` setter 里调用 `Parent?.OnPropertyChanged(nameof(TotalPrice))`，实现父级卡片总价实时更新。
- **ParentId 落库问题**：子行的 parent_id 指向主行 id，但 INSERT 时主行还没落库不知道 id。解决思路：① 两步事务（先插主行获取 id，再插子行）；② 或通过 GroupCode + PartName 关联查询还原。本期采用②（展示时通过 GroupCode + PartName 匹配；落库只存 GroupCode，ItemName 作为业务关联字段），后续可演进为两步事务。
- **SQLite ALTER TABLE**：SQLite 不支持 DROP COLUMN，但支持 `ALTER TABLE ADD COLUMN`。`Ensure*` 方法用 `IF NOT EXISTS` 防止重复执行。
