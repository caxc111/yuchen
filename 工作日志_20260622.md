## 今日完成工作

### 1. 产品物料表统一迁移

**问题背景**：
- 系统存在两套并行的产品物料存储方案：
  - `ProductPartMaterials` 表：早期实现，通过 `ProductPartMaterial` 模型类操作
  - `ProductMaterialLibrary` 表：后期设计，原生支持复合物料的父子关系（`parent_pml_id`）
- 两套方案并存导致维护成本增加，也给后续功能扩展带来隐患

**迁移目标**：
- 统一使用 `ProductMaterialLibrary` 表作为产品物料的唯一存储位置
- 统一使用 `SelectedMaterial` 模型类作为 UI 层与数据库之间的数据传输对象
- 简化 `DbService` 中的方法调用链，减少不必要的类型转换

**修改文件**：

| 文件 | 修改内容 |
|------|----------|
| `Models/SelectedMaterial.cs` | 添加 `Remarks`、`FactoryName` 字段，支持导出功能 |
| `Services/DbService.cs` | 修改 `LoadProductMaterialsFromLibrary()` SQL 查询，关联 FactoryProducts 表获取工厂名和备注 |
| `ViewModels/ProductManagementViewModel.cs` | `PersistSelectedMaterials()` 直接调用 `SaveProductMaterialsToLibrary()`，不再转换为 `ProductPartMaterial` |
| `Views/ProductManagementDialog.xaml.cs` | 加载时直接使用 `LoadProductMaterialsFromLibrary()`，保存时直接使用 `SaveProductMaterialsToLibrary()`，去掉中间转换 |
| `Views/ProductDetailsWindow.xaml.cs` | 改为调用 `LoadProductMaterialsFromLibrary()` |

**数据流对比**：

迁移前：
```
UI (SelectedMaterial) → 转换为 ProductPartMaterial → AddProductPartMaterials() → ProductPartMaterials 表
ProductPartMaterials 表 → GetProductPartMaterials() → 转换为 SelectedMaterial → UI
```

迁移后：
```
UI (SelectedMaterial) → SaveProductMaterialsToLibrary() → ProductMaterialLibrary 表
ProductMaterialLibrary 表 → LoadProductMaterialsFromLibrary() → UI (SelectedMaterial + Children)
```

**DbService 中相关方法**：
- `SaveProductMaterialsToLibrary(int productId, List<SelectedMaterial> materials)` - 保存物料（支持复合物料主行和子行）
- `LoadProductMaterialsFromLibrary(int productId)` - 加载物料（返回 `List<SelectedMaterial>`）
- `DeleteProductMaterialsLibrary(int productId)` - 级联删除

### 2. SelectedMaterial 模型扩展

为支持导出功能，添加以下字段：

```csharp
// 导出时需要的额外字段（从 FactoryProducts 关联获取）
public string Remarks { get; set; } = "";
public string FactoryName { get; set; } = "";
```

`LoadProductMaterialsFromLibrary()` SQL 查询也相应更新为 LEFT JOIN FactoryProducts 和 Factories 表：

```sql
SELECT pml.*,
       COALESCE(f.factory_name, '') as factory_name,
       COALESCE(fp.remarks, '') as remarks
FROM ProductMaterialLibrary pml
LEFT JOIN FactoryProducts fp ON pml.material_id = fp.id
LEFT JOIN Factories f ON fp.factory_id = f.id
WHERE pml.product_id = @productId
ORDER BY pml.pml_id
```
