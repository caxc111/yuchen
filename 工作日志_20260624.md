## 产品管理弹窗 - 户型字段自动更新修复

### 问题描述
编辑产品部件后，户型字段（HouseType）显示不正确。如：部件包含"主卧室*1, 主卫生间*1"，但户型仍显示"0R0B"。

### 根本原因
XAML 绑定的是 `Product.HouseType`，但代码中使用 `OnPropertyChanged(nameof(Product))` 触发通知时，UI 绑定的 `Product` 对象引用没变，所以不会刷新显示。

### 修复内容
1. 添加 `HouseTypeDisplay` 属性作为 UI 绑定的中转
2. 修改 XAML 绑定从 `Product.HouseType` 改为 `HouseTypeDisplay`
3. 更新所有设置户型的代码，改用 `OnPropertyChanged(nameof(HouseTypeDisplay))`

### 修改文件
- `FactoryProductManager/Views/ProductManagementDialog.xaml.cs`
  - 新增 `HouseTypeDisplay` 属性
  - 修改 `ContentRendered`、`UpdatePartsSummary` 等处的属性通知
- `FactoryProductManager/Views/ProductManagementDialog.xaml`
  - 绑定改为 `HouseTypeDisplay`

### 状态
已完成

## 窗口拖拽功能修复

### 问题描述
多个弹窗窗口无法拖动，包括：
- 工厂管理-编辑弹窗 (FactoryDialogWindow)
- 物料管理弹窗 (MaterialDialogWindow)
- ProductManagementDialog
- 以及所有其他无边框窗口

### 根本原因
1. `WindowDragBehavior.IsInteractiveElement` 方法将 `Border` 也列为交互元素，导致点击 Border 时跳过拖拽处理。
2. 部分窗口的 `WindowDragBehavior.Enable` 属性放置在嵌套的 Grid 上，而非 Window 的直接子元素。

### 修复内容
1. `App.xaml.cs`：从 `IsInteractiveElement` 方法中移除 Border 判断。
2. `FactoryDialogWindow.xaml`：将 `WindowDragBehavior.Enable` 从 Grid 移到 Border。
3. `MaterialDialogWindow.xaml`：将 `WindowDragBehavior.Enable` 从 Grid 移到 Border。
4. `ProductManagementDialog.xaml`：移除 Border 上的 `WindowDragBehavior.Enable`（全局样式已覆盖）。
5. `App.xaml`：移除不必要的全局 Window 样式。

### 涉及文件
- `FactoryProductManager/App.xaml.cs`
- `FactoryProductManager/Views/FactoryDialogWindow.xaml`
- `FactoryProductManager/Views/MaterialDialogWindow.xaml`
- `FactoryProductManager/Views/ProductManagementDialog.xaml`
- `FactoryProductManager/App.xaml`

### 状态
已完成

## 窗口拖拽拦截导航按钮修复

### 问题描述
`AddProductMaterialWindow` 等弹窗的左侧导航栏按钮点不动，无法切换部件。

### 根本原因
`WindowDragBehavior.Enable="True"` 被放在了内容 Border 或外层 Grid 上，导致整个窗口区域拦截了鼠标事件，连导航按钮都被拖拽事件吞掉了。

### 修复内容
将 `WindowDragBehavior.Enable` 从内容区域移到标题栏 Grid，确保只有标题栏可以拖拽，内容区域不受影响。

### 涉及文件
- `FactoryProductManager/Views/AddProductMaterialWindow.xaml`
- `FactoryProductManager/Views/FactoryDetailsWindow.xaml`
