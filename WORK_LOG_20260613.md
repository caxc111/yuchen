### 1. 修复窗口 TextBox/ComboBox 等交互元素无法获得焦点的问题

**问题背景**：
- 所有自定义弹窗（工厂、物料、产品详情等窗口）使用 `WindowDragBehavior` 的 `DragMove()` 实现窗口拖动。
- 原实现中，`PreviewMouseLeftButtonDown` 通过 `InputHitTest` 获取鼠标位置下的元素，再向上遍历 VisualTree 判断是否为交互元素。
- 问题在于：点击 ComboBox 时，`InputHitTest` 可能命中 ComboBox 内部的 Path/Border 等子元素，而非 ComboBox 本身，导致交互元素判断失败，触发了 DragMove()，使下拉被中断。

**解决方案**：
- 改用 `e.OriginalSource` 替代 `InputHitTest`，直接获取鼠标事件的原始触发元素，更准确。
- 增加 VisualTree 遍历深度从 20 层到 30 层，防止嵌套过深漏检。
- 新增 `ScrollBar`、`Slider` 到交互元素白名单，防止 DataGrid 滚动和滑块操作被 DragMove 劫持。
- 去除 `pos` 变量和无效的 `InputHitTest` 调用。

**修改前**：
```csharp
private static void Element_PreviewMouseLeftButtonDown(object sender, MouseButtonEventArgs e)
{
    if (e.ClickCount != 1 || sender is not UIElement element) return;
    var pos = e.GetPosition(element);
    var hit = element.InputHitTest(pos) as DependencyObject;
    if (hit != null && IsInteractiveElement(hit)) return;
    var window = GetWindow(element);
    window?.DragMove();
}
```

**修改后**：
```csharp
private static void Element_PreviewMouseLeftButtonDown(object sender, MouseButtonEventArgs e)
{
    if (e.ClickCount != 1 || sender is not UIElement element) return;
    var hit = e.OriginalSource as DependencyObject;
    if (hit != null && IsInteractiveElement(hit)) return;
    var window = GetWindow(element);
    window?.DragMove();
}
```

**交互元素白名单**（IsInteractiveElement 方法）：
- `ButtonBase`（按钮基类）
- `ComboBox`（下拉框）
- `ListBoxItem`（列表项）
- `CheckBox`（复选框）
- `RadioButton`（单选按钮）
- `TextBox` / `TextBoxBase`（文本框）
- `PasswordBox`（密码框）
- `ScrollBar`（滚动条）
- `Slider`（滑块）

**涉及文件**：
- `FactoryProductManager/App.xaml.cs`

### 2. 修复物料编辑保存后 DataGrid 页面不刷新的问题

**问题背景**：
- 在物料管理页面编辑已有物料并保存后，DataGrid 中的数据未更新，显示的仍是旧数据。
- 添加物料功能正常，因为添加后执行了 `Refresh()` 重新加载。
- 编辑后仅执行了 `UpdateMaterial()`，可能返回了新的对象引用但未触发 UI 刷新。

**根本原因**：
- `UpdateMaterial` 方法执行更新后，直接修改了传入的 `Material` 对象属性，但 DataGrid 绑定的 ObservableCollection 中引用的对象仍是同一个。
- 问题出在 `UpdateMaterial` 中使用了 `ReloadData()` 重新从数据库加载并替换集合元素，但 UI 绑定的通知机制未能正确触发。

**解决方案**：
- 修改 `UpdateMaterial` 方法，更新后改为重新调用数据库加载数据，替换集合中对应元素，与 `AddMaterial` 保持一致的处理方式。

**涉及文件**：
- `FactoryProductManager/ViewModels/MaterialViewModel.cs`

### 3. 新增工厂编码自动生成规则

**需求**：
- 工厂编码以 "S" 打头，后面为 3 位流水号（如 S001、S002、S003）。
- 点击"添加工厂"时，自动扫描数据库已有工厂编码，选择空余的最小流水号填入。

**实现方案**：
- 在 `DbService` 中新增 `GetNextFactoryCode()` 方法：
  - 查询所有 `factory_code LIKE 'S%'` 的记录。
  - 提取 S 后面的数字部分，存入 List。
  - 从 1 开始递增，找到第一个不在列表中的数字即为下一个可用编号。
  - 返回格式为 `S{num:D3}`（如 S001）。

- 在 `FactoryDialogUserControl` 构造函数中：
  - 判断是否为新增工厂（`factory == null`）。
  - 新增时调用 `_dbService.GetNextFactoryCode()` 获取编码。
  - 将编码赋值给 `Factory.FactoryCode`，通过绑定自动填入 TextBox。

**DbService 新增方法**：
```csharp
public string GetNextFactoryCode()
{
    using var conn = GetConnection();
    conn.Open();
    var cmd = new SQLiteCommand(@"
        SELECT factory_code FROM Factories
        WHERE factory_code LIKE 'S%'
        ORDER BY factory_code", conn);
    using var reader = cmd.ExecuteReader();
    var existingCodes = new List<int>();
    while (reader.Read())
    {
        var code = reader.GetString(0);
        if (code.Length > 1 && int.TryParse(code.Substring(1), out int num))
        {
            existingCodes.Add(num);
        }
    }
    int next = 1;
    while (existingCodes.Contains(next))
    {
        next++;
    }
    return $"S{next:D3}";
}
```

**涉及文件**：
- `FactoryProductManager/Services/DbService.cs`（新增 GetNextFactoryCode 方法）
- `FactoryProductManager/Views/FactoryDialogUserControl.xaml.cs`（构造时自动填入编码）

### 4. 修复物料选择弹窗（MaterialSelectorDialog）单击无法选中的问题

**问题背景**：
- 在 `MaterialSelectorDialog`（添加产品物料弹窗）中，DataGrid 绑定了一个 `IsSelected` 属性到业务对象 `FactoryMaterial`。
- 用户反馈必须双击才能选中和取消选中，单击无反应。

**排查过程**：
1. 添加详细日志发现 `SelectionChanged` 事件每次点击都在触发，`IsSelected` 属性确实被切换了。
2. 检查 XAML 样式发现 DataTrigger 绑定有问题。

**根本原因**：
- XAML 第 144 行 checkbox 的 DataTrigger 绑定到了 `DataGridRow.IsSelected`（DataGrid 内置选中状态），而非 `FactoryMaterial.IsSelected`（业务属性）：
  ```xml
  <!-- 错误写法：绑定到 DataGridRow 的 IsSelected（点击后被清空，永远为 false） -->
  <DataTrigger Binding="{Binding IsSelected, RelativeSource={RelativeSource AncestorType=DataGridRow}}" Value="True">
  ```
- 同时 RowStyle 中存在 `Trigger Property="IsSelected"`（DataGridRow 内置属性）与 `DataTrigger` 混用，导致样式冲突。

**解决方案**：
1. **修正 DataTrigger 绑定**：`{Binding IsSelected}` 直接绑定到业务对象的 `IsSelected` 属性，配合 `INotifyPropertyChanged` 自动触发 UI 刷新。
2. **移除冲突的 Trigger**：删除 `DataGridRow` 内置 `Trigger Property="IsSelected"`，保留 `DataTrigger` 作为唯一选中样式依据。
3. **改用 PreviewMouseLeftButtonDown 替代 SelectionChanged**：原来在 `SelectionChanged` 事件中处理单击，选中后立即 `SelectedItem = null`，这种方式依赖 DataGrid 内部选择机制，容易与 DataTrigger 冲突。改为 `PreviewMouseLeftButtonDown` 直接在鼠标按下时处理，通过 VisualTreeHelper 向上遍历找到 DataGridRow，获取其 DataContext（即 FactoryMaterial），切换 IsSelected，不依赖 DataGrid 的 Selection。

**修改后 XAML（RowStyle）**：
```xml
<Style.Triggers>
    <Trigger Property="IsMouseOver" Value="True">
        <Setter Property="Background" Value="{StaticResource WarmActionBrush}"/>
    </Trigger>
    <DataTrigger Binding="{Binding IsSelected}" Value="True">
        <Setter Property="Background" Value="#E3F2FD"/>
        <Setter Property="Foreground" Value="Black"/>
    </DataTrigger>
</Style.Triggers>
```

**修改后 XAML（CellTemplate checkbox DataTrigger）**：
```xml
<DataTemplate.Triggers>
    <DataTrigger Binding="{Binding IsSelected}" Value="True">
        <Setter TargetName="CheckBorder" Property="Background" Value="#2196F3"/>
        <Setter TargetName="CheckBorder" Property="BorderBrush" Value="#2196F3"/>
        <Setter TargetName="CheckMark" Property="Visibility" Value="Visible"/>
    </DataTrigger>
</DataTemplate.Triggers>
```

**修改后 C#（事件处理）**：
```csharp
private void MaterialsDataGrid_PreviewMouseLeftButtonDown(object sender, MouseButtonEventArgs e)
{
    var dep = e.OriginalSource as DependencyObject;
    while (dep != null && dep is not DataGridRow)
        dep = System.Windows.Media.VisualTreeHelper.GetParent(dep);
    if (dep is DataGridRow row && row.DataContext is FactoryMaterial material)
    {
        LogService.Debug($"[MaterialSelectorDialog] 单击选中: {material.MaterialName}");
        material.IsSelected = !material.IsSelected;
        UpdateOkButtonState();
    }
}
```

**最终交互逻辑**：
- **单击行** → 切换 IsSelected（打勾/取消勾选），点击"添加物料"按钮将选中项添加到产品
- **双击行** → 直接选中并关闭弹窗

**涉及文件**：
- `FactoryProductManager/Views/MaterialSelectorDialog.xaml`（RowStyle、CellTemplate、事件绑定）
- `FactoryProductManager/Views/MaterialSelectorDialog.xaml.cs`（事件处理方法）

## 技术要点

- `e.OriginalSource` vs `InputHitTest`：事件路由中，`OriginalSource` 返回实际触发事件的元素（鼠标/触摸），而 `InputHitTest` 可能命中命中测试覆盖的元素（视觉层渲染），前者更可靠。
- WPF 窗口拖动（DragMove）与交互元素冲突的通用解法：始终从 `OriginalSource` 向上遍历 VisualTree 判断是否交互元素。
- 编码流水号自动生成的核心算法：提取所有已有编号数字 → 排序 → 从 1 开始找第一个缺失值 → 考虑删除后复用编号的场景。
- DataGrid Row/Cell 选中样式的正确绑定：业务选中状态（`FactoryMaterial.IsSelected`）使用 `DataTrigger`；`DataGridRow` 内置 `IsSelected` 不要与 `DataTrigger` 混用，否则样式冲突。单击交互避免用 `SelectionChanged`（与 DataGrid 内部选择机制冲突），改用 `PreviewMouseLeftButtonDown` + `VisualTreeHelper.GetParent` 向上找到 DataGridRow 后直接操作业务对象。
- `VisualTreeHelper.GetParent` 遍历 WPF 视觉树：适用于需要从子元素向上找到父容器的场景（如从点击的单元格向上找到 DataGridRow）。
