## 二、数据库设计

### 2.1 数据库基本信息

- **数据库类型**：SQLite
- **数据库文件**：FactoryProductDB.db
- **数据库版本**：V3
- **连接字符串**：`Data Source=FactoryProductDB.db;Version=3;`

### 2.2 当前数据表结构

#### 表1：Factories（工厂表）

```sql
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    factory_type TEXT,
    address TEXT,
    certifications TEXT,
    description TEXT,
    scale TEXT,
    employee_count INTEGER,
    production_capacity TEXT,
    controlling_person TEXT,
    contact_person TEXT,
    contact_info TEXT,
    contact_method TEXT,
```

**API 接口设计**：

```
认证模块（/api/auth）
├── POST /api/auth/login          # 用户登录
├── POST /api/auth/register       # 用户注册
├── POST /api/auth/wechat-login   # 微信登录
└── POST /api/auth/refresh-token  # 刷新Token

工厂模块（/api/factories）
├── GET    /api/factories         # 获取工厂列表
├── GET    /api/factories/{id}    # 获取单个工厂
├── POST   /api/factories         # 创建工厂
├── PUT    /api/factories/{id}    # 更新工厂
└── DELETE /api/factories/{id}    # 删除工厂

产品模块（/api/products）
├── GET    /api/products          # 获取产品列表
├── GET    /api/products/{id}     # 获取单个产品
├── POST   /api/products          # 创建产品
├── PUT    /api/products/{id}     # 更新产品
└── DELETE /api/products/{id}     # 删除产品
```

## 八、数据库设计原则

### 8.1 命名规范

1. **表名**：PascalCase，单数形式
   - ✅ Factories
   - ❌ factory_list, FactoryTable

2. **字段名**：PascalCase
   - ✅ FactoryCode, FactoryName
   - ❌ factory_code, FACTORY_NAME

3. **外键**：表名 + Id
   - ✅ FactoryId, UserId
   - ❌ fid, uid

### 8.2 约束原则

1. **主键**：使用自增整数 id
2. **唯一性**：关键字段添加 UNIQUE 约束
3. **非空**：必填字段添加 NOT NULL
4. **外键**：使用 FOREIGN KEY 建立关联
5. **默认值**：合理设置 DEFAULT 值

### 8.3 索引设计

**建议添加索引的字段**：
- 外键字段（提升关联查询性能）
- 经常查询的字段（WHERE 条件）
- 排序字段（ORDER BY）
- 唯一约束字段（自动创建索引）

```sql
-- 示例：工厂编码索引
CREATE INDEX idx_factories_code ON Factories(factory_code);

-- 示例：产品外键索引
CREATE INDEX idx_products_factory ON FactoryProducts(factory_id);
```
