# 工作日志 - 2026年6月17日

## 完成的工作

### 1. 导出Excel工厂名称字段修复
**问题**：硬装一级产品明细导出时，G列"工厂名称"为空，但H列"工厂物料编码"有值。

**原因**：`GetProductPartMaterials` 查询方法没有 JOIN 工厂表，导致无法获取工厂名称。

**修复**：
- 修改 `DbService.cs` 中 `GetProductPartMaterials` 方法，添加 LEFT JOIN 关联 `FactoryProducts` → `Factories`
- 在 `ProductPartMaterial` 模型中添加 `FactoryName` 属性
- 更新导出代码 `ProductManagementViewModel.cs`，G列使用 `mat.FactoryName`

### 2. 添加产品对话框字段隐藏
**需求**：在添加产品阶段，隐藏"面积（㎡）"、"成本合价（元）"、"销售合价（元）"三个字段。

**修复**：
- 添加 `InverseBooleanToVisibilityConverter` 转换器
- 在 `ProductManagementDialog.xaml.cs` 中添加 `IsAdding` 属性
- 为三个字段的 Grid 添加 `Visibility="{Binding IsAdding, Converter={StaticResource InverseBoolToVisibilityConverter}}"` 绑定
- 添加产品时隐藏，编辑产品时显示

### 3. 平面图字段改为图片上传
**需求**：平面图文本框改为支持点击选择或拖拽上传图片。

**参考**：参考了物料管理 `MaterialDialogUserControl.xaml` 的图片处理代码。

**修复**：
- 修改 `ProductManagementDialog.xaml`，将平面图文本框替换为图片拖拽/选择区域
- 添加 `FloorPlanDropZone` Border，支持 `DragOver`、`Drop`、`Click` 事件
- 添加 `FloorPlanPreviewImage` 用于预览
- 在代码后置中添加图片处理方法：
  - `FloorPlanDropZone_DragOver` - 拖拽悬停
  - `FloorPlanDropZone_Drop` - 拖拽放下
  - `FloorPlanDropZone_Click` - 点击选择
  - `IsValidImageFile` - 验证图片格式
  - `LoadFloorPlanImage` - 加载并保存图片到 Images 文件夹
  - `SetFloorPlanPreviewImage` - 显示预览
- 添加编辑产品时加载已有平面图的逻辑

**支持格式**：jpg, jpeg, png, gif, bmp
