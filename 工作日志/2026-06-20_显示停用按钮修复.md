# 2026-06-20 产品管理界面优化

## 修复的问题

### 1. "显示停用"按钮消失问题

**问题描述**：
- 有停用产品时，"显示停用"按钮不显示
- 所有产品都停用时按钮也不显示
- 点击按钮后文字没有变化

**根本原因**：
1. `Product`类没有实现`INotifyPropertyChanged`接口
2. 修改`IsActive`属性后不会触发属性变更通知
3. ViewModel中的`HasInactiveProducts`等计算属性无法重新评估
4. `HasInactiveProductsVisibility`绑定使用了错误的`BooleanToVisibilityConverter`转换器（该属性已返回`Visibility`类型）

**修复内容**：

1. **Product.cs** - 实现属性变更通知
   - 让Product类继承`INotifyPropertyChanged`
   - 所有属性改为属性语法，自动触发`OnPropertyChanged`
   - 修改`IsActive`时会通知所有依赖属性

2. **ProductManagementViewModel.cs** - 完善属性通知
   - 在`LoadProducts`方法中添加了`ShowInactiveButtonText`、`HasInactiveProducts`、`HasInactiveProductsVisibility`的通知

3. **ProductManagementView.xaml** - 修复绑定问题
   - 添加`BooleanToVisibilityConverter`资源
   - 移除按钮上多余的`Converter`绑定（`HasInactiveProductsVisibility`已返回`Visibility`）
   - 将Visibility绑定移到Border容器上
   - 移除按钮的固定宽度限制，让文字自适应

**关键代码变更**：

```csharp
// Product.cs - 实现属性变更通知
public class Product : INotifyPropertyChanged
{
    public event PropertyChangedEventHandler? PropertyChanged;
    private bool _isActive;

    public bool IsActive
    {
        get => _isActive;
        set { _isActive = value; OnPropertyChanged(nameof(IsActive)); }
    }
}
```

```xml
<!-- ProductManagementView.xaml - 修复绑定 -->
<BooleanToVisibilityConverter x:Key="BoolToVisibilityConverter"/>
<Border Visibility="{Binding HasInactiveProductsVisibility}">
    <Button Content="{Binding ShowInactiveButtonText}" .../>
</Border>
```
