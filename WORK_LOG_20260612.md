## 今日完成工作

### 1. 将监控日志（debug.log）移至项目根目录

**问题背景**：
- 原来 `debug.log` 生成在 `bin/Debug/net10.0-windows/` 目录下，位置较深不易查找。
- 用户希望日志放在项目根目录便于访问。

**解决方案**：
- 修改 `LogService.RootDebugLogPath` 属性，改为从 `AppDomain.CurrentDomain.BaseDirectory` 向上回溯 5 层目录，以定位到项目根目录。

**修改前**：
```csharp
public static string RootDebugLogPath => Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "debug.log");
```

**修改后**：
```csharp
public static string RootDebugLogPath
{
    get
    {
        string baseDir = AppDomain.CurrentDomain.BaseDirectory;
        for (int i = 0; i < 5 && !string.IsNullOrEmpty(baseDir); i++)
        {
            baseDir = Path.GetDirectoryName(baseDir) ?? string.Empty;
        }
        return Path.Combine(baseDir, "debug.log");
    }
}
```

**涉及文件**：
- `FactoryProductManager/Services/LogService.cs`

### 2. 调查编辑物料弹窗工厂/品牌回填问题

**问题现象**：
- 编辑已有物料时，品牌下拉框未正确显示原有品牌。

**调试日志分析**：
通过在关键方法入口添加日志，发现如下调用时序问题（时间戳为关键节点）：

```
17:53:03.892  [MaterialDialogUserControl] Material.FactoryName='佛山市南海区和邦盛世建材有限公司', Material.FactoryId=2
17:53:03.897  [ResetFactoryComboBoxItems] factoryList.Count=0, Material.FactoryName BEFORE=佛山市南海区和邦盛世建材有限公司
17:53:03.913  [InitializeFactories] ...
```

**根本原因定位**：
- `SetCurrentCategory` 在构造函数末尾被调用（第 81 行，位于 `InitializeFactories` 之前）。
- `SetCurrentCategory` 设置 `CategoryLevel1.SelectedIndex` 时，触发了 `ComboBox.SelectionChanged` 事件。
- 该事件处理器 `CategoryLevel1_SelectionChanged` → `UpdateFactoryOptionsByLevel1` → `ResetFactoryComboBoxItems(Enumerable.Empty<Factory>())` 在 `InitializeFactories` 之前执行。
- `ResetFactoryComboBoxItems` 收到空列表时，将 `Material.FactoryName` 清空。
- 同时 `ResetBrandComboBoxItems(Enumerable.Empty<Factory>())` 也被调用，清空了品牌下拉框，导致品牌无法匹配。

**修复尝试**：
- 调整构造函数中初始化顺序，将 `InitializeFactories()` 移至 `SetCurrentCategory()` 之后。
- 期望通过在 `SetCurrentCategory` 触发 SelectionChanged 之前已设置好 `_savedFactoryName`，使得 `UpdateFactoryOptionsByLevel1` 中的 `skipReset` 逻辑能保护工厂/品牌不被清空。
- 修复后编译验证通过，但用户反馈问题仍未完全解决，需进一步调查。

**涉及文件**：
- `FactoryProductManager/Views/MaterialDialogUserControl.xaml.cs`
- `FactoryProductManager/Services/LogService.cs`

## 技术要点

- 在 WPF 中，ComboBox 的 `SelectedIndex` 赋值会同步触发 `SelectionChanged` 事件，在构造函数初始化阶段尤其需要注意顺序。
- `skipReset` 参数用于在已知编辑已有数据的情况下跳过清空操作，但其生效前提是 `_savedFactoryName` 已被正确设置。
- `ResetFactoryComboBoxItems` 收到空集合时，会将 `Material.FactoryName` 置空，这是导致工厂名和品牌丢失的直接原因。
- 中文路径对 `Path.GetDirectoryName` 的影响需要通过增加回溯层数（5 层）来规避。
